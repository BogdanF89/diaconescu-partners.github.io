# Legal news aggregator

The "Legal News" section (`#news` in `index.html`) is populated from `news.json`, which is generated by an RSS aggregator script. Nothing runs in the visitor's browser except a relative `GET news.json`.

> For how this behaves when moving to a custom domain, see [`domain-migration.md`](domain-migration.md).

## Three independent pieces
1. **Aggregator** — `scripts/fetch-news.mjs`. Fetches RSS/Atom feeds, filters by keyword, merges with previous results, writes `news.json`.
2. **Static file** — `news.json` at the repo root, deployed alongside `index.html`.
3. **Renderer** — JS in `index.html` does an XHR `GET news.json` and builds the marquee cards.

## How it runs
- **Automatically:** `.github/workflows/news.yml` (cron `0 */6 * * *`, i.e. every 6 hours) on GitHub's servers, Node 20. It runs the script and commits an updated `news.json` to `main`.
- **Manually (workflow):** trigger `news.yml` via *workflow_dispatch* in the Actions tab.
- **Locally:** `npm install && npm run fetch-news` (writes `news.json` in the working directory).

## Editing sources
Sources live in the `SOURCES` array near the top of `scripts/fetch-news.mjs`:
```js
const SOURCES = [
  { label: 'Juridice.ro',       url: 'https://www.juridice.ro/feed/' },
  { label: 'Universul Juridic', url: 'https://www.universuljuridic.ro/feed/' },
  { label: 'Ziarul Financiar',  url: 'https://www.zf.ro/rss/' },
  { label: 'Economica.net',     url: 'https://www.economica.net/rss/' },
];
```
- `label` is shown on the card as the source tag.
- `url` must point to a working RSS/Atom feed.
- Add or remove entries freely; a feed that fails is logged as `[warn]` and skipped (it does not break the run).

## Editing keywords
An item is kept only if its title/description/categories contain one of the `KEYWORDS`. Matching is **case-insensitive and diacritic-insensitive** (`norm()` strips diacritics), so list both forms is for readability, not strictly required — but listing the diacritic-free form alone would still match.

Keep keywords **specific**. Generic words cause false positives: a bare `comercial` once matched an unrelated "spații comerciale" fire article, which is why the list now uses phrases like `drept comercial`, `litigiu comercial`, `litigii comerciale` and `drept civil`, `litigiu civil`, `litigii civile` instead.

## Tuning knobs (constants in the script)
- `MAX_ITEMS = 12` — total cards written to `news.json`
- `MAX_PER_SOURCE = 6` — cap per feed before merging
- `EXCERPT_LEN = 180` — excerpt truncation length
- `FETCH_TIMEOUT_MS = 20000` — per-feed timeout

## Behaviour notes
- **Merge & backfill:** fresh items take priority; previously-stored items (from the existing `news.json`) are kept if still unique by `link`. This means a temporarily-down feed doesn't wipe the section.
- **Sort:** newest first by `date`, then sliced to `MAX_ITEMS`.
- **Output shape:**
  ```json
  { "generatedAt": "ISO", "count": 12, "items": [
    { "source": "...", "title": "...", "link": "...", "date": "ISO", "excerpt": "..." }
  ] }
  ```
- The script calls `process.exit(0)` at the end on purpose — a hung feed connection can otherwise keep Node's event loop open indefinitely.

## Dependencies
Only `rss-parser` (see `package.json`). It is used **only** by this script — the website itself ships no Node dependencies.
